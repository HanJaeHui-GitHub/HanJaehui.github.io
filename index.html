1. Problem Definition – Data Reality

  • 추천을 위한 행동 로그(Click/View) 부재, 오직 '접수' 데이터만 존재
  • "데이터가 없어서 AI가 불가능하다"는 기술적 한계를 넘어, "탐색 비용을 줄여 이탈을 막는다"는 비즈니스 목표로 문제 재정의했습니다.
0% 비회원

2. Phase 1 – 상세 세그먼트 로직

복잡한 딥러닝 대신, 마케터가 이해하고 운영할 수 있는 '설명 가능한 로직'으로 즉시 도입 가능한 모델을 구현했습니다.

  • GUEST (92%)
    개인 식별 정보 부재 → 로그 기반 인기(Log)와 누적 조회수(Hit) 결합
    Score = 0.7 * log(Logs) + 0.3 * log(Hits)
  • DORMANT (7.7%)
    장기 미접속 → EASY(난이도) 가중치 2배 (복귀 장벽 완화)
    EASY 분류: 제목/분야에 ['네이밍','슬로건','사진','퀴즈','아이디어'] 포함 시
    Score = Norm(Popularity) * Difficulty_Weight (EASY=2.0)
  • LIGHT (소수)
    관심사 보유 → 콘텐츠 필터링(CBF) (태그 매칭)
    Score = 1.0 * Interest_Match(TF-IDF) + 0.3 * Popularity
  • POWER (0.18%)
    접수 2회+ → 이력 기반 하이브리드 (정밀 개인화)
    Score = 1.5 * History_Sim + 1.0 * Interest + 0.3 * Popularity

3. Phase 2 – 데이터 자산화 설계

AI 성능의 한계는 알고리즘이 아닌 데이터 구조에 있음을 설득하며, DB 설계를 역제안했습니다.

  • Implemented (데모/통합 기준): 비회원 식별을 위해 NM25_00001 형식의 가상 PK 체계를 설계하고, 비회원 접수 이력을 사용자 단위로 통합했습니다.
  • Implemented (설계 산출물): 관리자 입력을 Text → Code로 정형화하는 스키마 전환안을 제시했습니다. (예: age_min=19, region_code=11)
  • Proposed (고도화 제안): User_Activity_Log 테이블을 신설해 Result(접수)뿐 아니라 Process(노출/클릭) 로그를 축적하는 파이프라인을 제안했습니다.
Before
"서울 거주 20대..."
(비정형 텍스트)
→
After
age:19, region:11
(구조화된 코드)

1. GIS 정합성 확보

  • Kakao API Geocoding: 위경도 변환
  • 법정동/행정동 불일치 해결: GeoJSON MultiPolygon 기반 좌표 포함 여부 판별
  • 정밀 매핑: 57,790건 데이터를 행정동 단위로 정제

2. 인자 분석 (Factor Analysis)

단순 위험 점수가 아닌 위험의 원인(Why)을 설명하기 위해 인자 분석을 적용했습니다.

  • 통계 검증: KMO = 0.6078, Bartlett p < 0.05
  • 5대 잠재 인자: 고령화·독거, 경제 활동, 인지 건강, 금융 취약, 생활 인프라

3. 공간 분석 및 군집화

  • Moran's I: 위험 지역의 공간 군집(Hotspot) 확인 및 권역 기반 관리 필요성 제시
  • K-Means (k=3): 고위험 / 중위험 / 저위험 그룹 분류
