Problem Definition: Data Reality
  • 추천을 위한 행동 로그(Click/View) 부재, 오직 '접수' 데이터만 존재
  • 추천의 목적을 “정교한 개인화”보다 "탐색 비용을 줄여 이탈을 막는다"는 비즈니스 목표로 문제 재정의
  • 따라서 ‘개인화 모델’이 아닌, 데이터 밀도에 따라 목표가 다른 세그먼트 전략이 최적
92% 비회원
8% 회원
PHASE 1
Segment Strategy

마케터가 이해하고 운영할 수 있는 '설명 가능한 로직'으로 즉시 도입 가능한 모델을 구현했습니다.

세그먼트별 목표를 ‘탐색 유도–재진입–취향 학습–정밀 개인화’로 분리해, 동일한 데이터 현실에서도 추천이 작동하도록 설계했습니다.

GUEST (92%) 탐색 유도
인기 + 최신성 반영
Score = 0.7 · Norm(log(1 + Pop)) + 0.3 · Norm(RecencyDecay)
Pop=최근 N일 인기, RecencyDecay=exp(-d/τ)
DORMANT (7.7%) 장벽 완화
EASY 공모전 가중치
Score = Norm(Pop) · EasyWeight
EASY=[네이밍, 사진, 퀴즈]
LIGHT (소수) 취향 매칭
관심사 기반 CBF
Score = Norm(Σ (IDF(tag) · match(tag)))
match=관심사 일치(0/1)
POWER (0.18%) 정밀 개인화
이력 기반 Hybrid
Score = 1.5 · Norm(SimilarityCF) + 1.0 · Norm(Σ (IDF(tag) · match(tag)))
Sim=Cosine, Interest=Tag Match
PHASE 2
Data Assetization

AI 고도화의 병목을 데이터 구조로 정의하고, 운영 DB를 ‘학습 가능한 자산’으로 전환하는 설계를 역제안했습니다.

👤 식별 (Identity) 회원/비회원 통합 PK 체계(NM25_...)로 사용자 단위 데이터 연결
⚡ 신호 (Signal) 접수(Result)뿐 아니라 노출/클릭 등 과정(Process) 로그 축적 파이프라인
📏 표준 (Standard) 입력값·용어를 코드(Code)로 통일해 데이터 품질을 구조적으로 보장
💡 Impact: Result 중심 → Process 중심으로 추후 ML/DL 확장 가능한 데이터 인프라 전환

AI를 단순 기능이 아닌 운영 인프라 관점으로 연결했습니다."

1. Problem & Logic

  • 단순히 "위험하다"는 결과보다 "왜 위험한가?"를 설명할 수 있는 근거가 필수적이었습니다.
  • GIS 정밀 가공: 주소 데이터 57,790건을 위경도로 변환하고, GeoJSON 매핑 함수를 직접 구현하여 행정동 단위의 데이터 정확도를 확보했습니다.

2. Analysis Framework

  • 인자 분석 (Factor Analysis): 14개의 파생 지표를 ①고령화/독거 ②경제력 ③인지건강 ④금융취약 ⑤생활인프라 등 5개의 해석 가능한 핵심 인자로 압축했습니다. (KMO 0.6078 검증)
  • 공간 분석 (Spatial Analysis): Moran's I를 통해 위험 지역의 공간적 군집(Hot Spot)을 도출하여 관리의 우선순위를 정했습니다.
  • 군집화 (K-Means): 3가지 위험 유형(복합 결핍, 인프라 결핍, 소득 결핍)으로 분류했습니다.
