PHASE 1
Segment Strategy

마케터가 이해하고 운영할 수 있는 설명 가능한 로직으로 즉시 도입 가능한 모델을 구현했습니다.

GUEST (92%) 탐색 유도
인기 + 최신성 반영
Score = 0.7 · Norm(log(1 + Pop)) + 0.3 · Norm(RecencyDecay)
Pop=최근 N일 인기, RecencyDecay=exp(-d/τ)
DORMANT (7.7%) 장벽 완화
EASY 공모전 가중치
Score = Norm(Pop) · EasyWeight
EASY=[네이밍, 사진, 퀴즈]
LIGHT (소수) 취향 매칭
관심사 기반 CBF
Score = Norm(Σ (IDF(tag) · match(tag)))
match=관심사 일치(0/1)
POWER (0.18%) 정밀 개인화
이력 기반 Hybrid
Score = 1.5 · Norm(SimilarityCF) + 1.0 · Norm(Σ (IDF(tag) · match(tag)))
Sim=Cosine, Interest=Tag Match
PHASE 2
Data Assetization

AI 고도화의 병목을 데이터 구조로 정의하고, 운영 DB를 학습 가능한 자산으로 전환하는 설계를 역제안했습니다.

👤 식별 (Identity) 회원/비회원 통합 PK 체계(NM25_...)로 사용자 단위 데이터 연결
⚡ 신호 (Signal) 접수(Result)뿐 아니라 노출/클릭 등 과정(Process) 로그 축적 파이프라인
📏 표준 (Standard) 입력값·용어를 코드(Code)로 통일해 데이터 품질을 구조적으로 보장

위험유형별 처방 설계

1. Problem & Logic

  • 단순히 “위험하다”는 결과보다 “왜 위험한가?”를 설명할 근거가 필수였습니다.
  • GIS 정밀 가공: 주소 데이터 57,790건을 위경도로 변환하고, GeoJSON 매핑 함수를 직접 구현해 행정동 단위 정확도를 확보했습니다.

2. Analysis Framework

  • 인자 분석: 14개 파생 지표를 5개 해석 가능한 인자(고령화/독거, 경제력, 인지건강, 금융취약, 생활인프라)로 압축 (KMO 0.6078)
  • 공간 분석: Moran’s I로 군집(Hot Spot)을 확인해 관리 우선순위를 설정
  • 군집화: K-Means로 3가지 위험유형(복합 결핍/인프라 결핍/소득 결핍) 도출

3. Result & Policy Proposal
